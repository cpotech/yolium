# Scheduled Agent Edit Workflow

## What was implemented

- Added an in-place edit path for existing scheduled agents by reusing `AddSpecialistDialog` in edit mode.
- Added a main-process `schedule:update-definition` IPC path backed by a shared create/update specialist writer that validates markdown and keeps the original specialist id locked in frontmatter.
- Added an `Edit definition` action to the specialist config dialog and wired `SchedulePanel` to open the shared editor from the configure flow, then refresh state after save.
- Changed specialist credential persistence to merge partial per-service updates so existing secrets are preserved.
- Documented the edit workflow in `docs/CRON-AGENTS.md`.

## Files modified and why

- `src/main/services/specialist-scaffold.ts`: shared validated file writer for create and update operations.

- `src/main/stores/specialist-credentials-store.ts`: merge-safe credential persistence.
- `src/renderer/components/schedule/AddSpecialistDialog.tsx`: edit-mode preload/reset logic, locked ids, save flow, and create/edit UI differences.

- `src/renderer/components/schedule/SchedulePanel.tsx`: configure-to-edit handoff and editor close/reload flow.
- `docs/CRON-AGENTS.md`: documented editing existing scheduled agents and merge-safe credential updates.
- `src/tests/specialist-scaffold.test.ts`: overwrite/update writer coverage.
- `src/tests/AddSpecialistDialog.test.tsx`: edit-mode preload, locked-name, and update-vs-create coverage.

## Caveats / follow-up

- The Playwright Electron run in this shell needed a manual Xvfb display (`DISPLAY=:99`) because the default headless launch path could not open a window.
- The E2E spec copies the real cron specialist markdown files into `<repo>/agents/cron` before launch because the built main-process loader resolves specialists from that packaged path.
