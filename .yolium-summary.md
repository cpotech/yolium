## What was implemented

Implemented specialist creation UI in the schedule panel:
- Added a new `AddSpecialistDialog` modal to collect specialist name (required) and description (optional).
- Added kebab-case sanitization for specialist names on blur and before submit.
- Wired dialog submit to existing `window.electronAPI.schedule.scaffold()` IPC bridge.
- Added inline error display for scaffold failures (including duplicate specialist name errors).
- Added close behavior for Escape key, backdrop click, and Cancel button.
- Added `+ Add` action in the SchedulePanel header and replaced empty-state guidance with an `Add Specialist` CTA.
- Wired panel callback to reload schedule state and close the dialog after successful creation.
