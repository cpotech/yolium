# Twitter Posting And Action Tracking

## What was implemented

- Added `action` protocol support and fixed `run_result` parsing so scheduled agents can emit both result summaries and action telemetry without being dropped.
- Added a new JSONL-backed action log store and wired scheduled runs to persist parsed action messages with run and specialist metadata.
- Exposed action history and action stats through schedule IPC/preload APIs and rendered them in the schedule UI:
  - specialist cards now show an `Actions` stat
  - run detail now shows an `Actions` section between `Summary` and `Agent Log`
- Added stdlib-only Twitter helper scripts for posting tweets with OAuth 1.0a and reading mentions with bearer-token auth, then mounted them into the Docker image.
- Updated `twitter-growth.md` with the missing write credentials plus dry-run, helper-usage, and `@@YOLIUM action` emission instructions.
