# Yolium Plan: "(bug) q should close git diff dialog"

## Context

The GitDiffDialog component currently only responds to `Ctrl+Q` (or `Cmd+Q` on macOS) to close the dialog. The bug is that pressing just `q` (without modifier keys) should also close the dialog.

## Analysis

**Relevant Files:**
- `src/renderer/components/code-review/GitDiffDialog.tsx:210-218` - Uses `isCloseShortcut(e)` which only handles Ctrl+Q/Cmd+Q
- `src/renderer/lib/dialog-shortcuts.ts:4-6` - Defines `isCloseShortcut` function
- `src/renderer/components/code-review/GitDiffDialog.tsx:242` - Shows shortcut hint "Ctrl+Q"
- `src/tests/dialog-close-shortcut.test.ts` - Existing unit tests for close shortcut

**Current Behavior:**
- `isCloseShortcut` returns `true` only when `(e.ctrlKey || e.metaKey) && e.key.toLowerCase() === 'q'`
- The GitDiffDialog imports and uses this function in its `handleKeyDown` callback

**Bug:**
- Pressing plain `q` key does not close the dialog
- Only Ctrl+Q or Cmd+Q works

## Approach

Add a check for plain 'q' key in the GitDiffDialog's keyboard handler. This is the safest approach because:
1. It doesn't change behavior for other dialogs that rely on `isCloseShortcut`
2. The bug specifically mentions the git diff dialog

**Changes needed:**
1. Modify `GitDiffDialog.tsx` to handle plain 'q' key in `handleKeyDown`
2. Update the shortcut hint in the UI from "Ctrl+Q" to "q" 
3. Add unit tests for the new keyboard behavior
4. Add E2E test to verify 'q' closes the dialog

## Implementation Steps

### Step 1: Modify GitDiffDialog.tsx

In `src/renderer/components/code-review/GitDiffDialog.tsx`:
- Line 210-218: Update `handleKeyDown` to also check for plain 'q' (without Ctrl)
- Line 242: Update the shortcut hint from "Ctrl+Q" to "q"

### Step 2: Add Unit Tests

Create `src/tests/git-diff-dialog-shortcuts.test.tsx`:
- Test that pressing 'q' key closes the dialog
- Test that pressing 'Q' (uppercase) key also closes the dialog
- Test that Ctrl+Q still works

### Step 3: Add E2E Test

Add to `src/tests/e2e/tests/` (e.g., `dialog-shortcuts.spec.ts` or new file):
- Test that pressing 'q' closes the git diff dialog when open
