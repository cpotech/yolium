# Implementation Plan: Log panel `l` toggle + `j`/`k` navigation

## Context

Two related features for the agent log panel in ItemDetailDialog:

1. **Toggle shortcut** â€” Add `l` in sidebar zone (NORMAL mode) to expand/collapse the agent log panel. Currently only mouse-click works.
2. **In-log navigation** â€” When the log panel is open, `j`/`k` should navigate (scroll) within the log content. Currently auto-scroll to bottom always wins on new output.

## Analysis

- `useAgentSession` manages `showAgentLog` state (line 46) via internal `setShowAgentLog` â€” needs a `toggleAgentLog` action exposed
- `AgentLogPanel` has a `logContainerRef` pointing to the scrollable content div (`data-testid="agent-log-content"`)
- Auto-scroll in `AgentLogPanel` (line 31-35) always scrolls to bottom on new output â€” needs a flag to pause auto-scroll when user navigates
- `ItemDetailDialog` has a `focusZone` state (`'editor' | 'sidebar'`) â€” needs a third `'log'` value
- Sidebar zone shortcuts are in `handleKeyDown` (lines 246-272)
- `vim-actions.ts` central manifest â€” both shortcuts must be listed
- `AgentLogPanel` manages its own internal `isExpanded` state â€” it needs to be controlled from outside

## Approach

1. Expose `toggleAgentLog` from `useAgentSession` â€” allows opening the log from keyboard
2. Refactor `AgentLogPanel` to accept `isExpanded` as a controlled prop (from parent) + expose `logContainerRef` via `useImperativeHandle` so parent can control scroll
3. Add `logScrollPausedRef` and `logFocusedLineIndex` state in `ItemDetailDialog` to track manual navigation within the log
4. Add `focusZone: 'log'` state â€” when active, `j`/`k` scroll within log content instead of sidebar
5. When `j`/`k` are pressed in sidebar and log is open, enter log focus mode
6. Register both shortcuts in `vim-actions.ts`

## Steps

### Step 1: Expose `toggleAgentLog` from `useAgentSession`
**File:** `src/renderer/hooks/useAgentSession.ts`
- Add `toggleAgentLog` to `AgentSessionActions` interface (line 20)
- Implement: `toggleAgentLog = useCallback(() => setShowAgentLog(prev => !prev), [])`
- Add to returned object (line 270)

### Step 2: Refactor `AgentLogPanel` to controlled `isExpanded`
**File:** `src/renderer/components/agent/AgentLogPanel.tsx`
- Accept `isExpanded?: boolean` prop (controlled) + `onToggle?: () => void`
- When `isExpanded` is provided, use it instead of internal state; call `onToggle` on click
- Expose `logContainerRef` and `pauseAutoScroll()` / `resumeAutoScroll()` via `useImperativeHandle` on a new ref type
- When `pauseAutoScroll()` is called, disable auto-scroll in the `useEffect` (add a `autoScrollPausedRef`)
- When `resumeAutoScroll()` is called, re-enable and scroll to bottom

### Step 3: Update ItemDetailDialog state and refs
**File:** `src/renderer/components/kanban/ItemDetailDialog.tsx`
- Add `logPanelRef` using `useRef` with the imperative handle type from AgentLogPanel
- Add `logFocused` state: `useState<boolean>(false)` â€” tracks whether log panel has keyboard focus
- Add `logLineCount` state to track output line count for detecting new content
- Update `AgentLogPanel` render to pass `isExpanded={agentSession.showAgentLog}` and `ref={logPanelRef}`

### Step 4: Add `j`/`k` log navigation handlers
**File:** `src/renderer/components/kanban/ItemDetailDialog.tsx`
- When `logFocused` is true and `j`/`k` is pressed in NORMAL mode:
  - `j`: scroll log container down by ~3 lines (using `scrollBy`), call `logPanelRef.current?.pauseAutoScroll()`
  - `k`: scroll up by ~3 lines, call `pauseAutoScroll()`
  - Auto-scroll resumes when user presses `l` to collapse or `Escape` to exit log focus
- When log is open and user presses `j`/`k` while in sidebar zone, automatically enter log focus
- `Escape` in log focus mode: exit log focus, resume auto-scroll, return focus to dialog container

### Step 5: Add `l` shortcut in sidebar zone
**File:** `src/renderer/components/kanban/ItemDetailDialog.tsx`
- In `handleKeyDown` sidebar zone block (line 246), add: `if (event.key === 'l') { event.preventDefault(); agentSession.toggleAgentLog(); return }`
- When `l` collapses the log, also exit log focus mode if active
